=============== SYSTEM DESIGN PRINCIPLES TO FOLLOW ======================
Frontend Ideas:

- premium designs with best look and feel
- code splitting
- API Optimization
- Memoize components when needed
- use useMemo, useCallbacks
- Use react-virtual (tanstack-virtualization) whereever needed
- define custom hooks wherever needed
- Reduce JS size , Avoid heavy libraries, Import only what you use
- Image Optimization
- Debounced search
- define Reusable components
- Error Handling (use toasts)
- Empty states No data вҶ’ show illustration + CTA
  -Smooth transitions
  -No layout shifts
  -Consistent button styles
  -Proper loading states
  -Fast feedback on actions

рҹ‘ү Fast > fancy
рҹ‘ү Consistency > creativity
рҹ‘ү Simplicity > complexity

Backend Ideas:

1. DRY
2. KISS
3. YAGNI
4. try to push the non-critical tasks (reporting, analytics, etc) to queue
5. Focus on very quick responding system, try to use caching
